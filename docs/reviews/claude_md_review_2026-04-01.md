## 问题一（主要）：§6 规范 与 §10 禁止事项 高度重叠

§10"代码规范"子节共 11 条，其中 **8 条是 §6 已有规则的纯负向重复**：

| §10 禁止条目 | §6 对应位置 |
|-------------|------------|
| 禁止在 `Mapped[]` 中使用 SQLAlchemy 类型 | §6 ORM 模型（代码示例已标注 ❌） |
| 禁止在测试文件硬编码真实密码 | §6 安全规范 |
| 禁止在模块顶层注册测试路由 | §6 测试 |
| 禁止在 Service/Route 层直接操作 ORM | §6 数据库 |
| 禁止在 Adapter 层直接调用 SDK | §6 数据采集层 |
| 禁止在集成测试使用连接池 | §6 数据库 |
| 禁止在 Engine 层进行 IO | §6 Engine 层 |
| 禁止在 `backend/` 以外执行 alembic | §8 迁移文件规范 |

剩余 3 条是 §6 中没有的**唯一性**禁令：

- `禁止跳过 TDD 流程直接写实现`
- `禁止使用 declarative_base() 函数`
- `禁止使用 tool.uv.dev-dependencies`

### 建议修改

1. **删除** §10"代码规范"子节全部 11 条
2. 将上述 3 条唯一性禁令**合并入 §6** 各自对应的小节：
   - `禁止跳过 TDD` → §5 或 §6 测试小节
   - `禁止 declarative_base()` → §6 ORM 模型小节
   - `禁止 tool.uv.dev-dependencies` → §6（新增"包管理"小节，或附于 §4 命令）
3. §10 仅保留"Phase 文档治理"4 条，节标题可更名为"Phase 文档治理规则"
