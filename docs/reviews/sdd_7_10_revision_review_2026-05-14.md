## 1. 关键问题（必须处理 / Blocker）

### B-1 ❌ §7.6 综合评分公式存在方差归一化缺失（数学硬错误）

**草案 §2.5 公式：**
```
composite_z(stock) = Σ_i w_i(state) × strategy_z_i(stock)
composite_score_0_100(stock) = Φ(composite_z) × 100
```

**问题：** 若 strategy_z_i ~ N(0,1) 且 4 个策略经正交化后互相独立，则：
```
Var(composite_z) = Σ w_i² × Var(strategy_z_i) = Σ w_i²
```

以默认权重 w = (0.40, 0.25, 0.15, 0.20)：Σ w_i² = 0.16 + 0.0625 + 0.0225 + 0.04 = **0.285**
→ Std(composite_z) ≈ 0.534（**远小于 1**）
→ composite_z 实际服从 N(0, 0.285)，**不是标准正态**
→ Φ(composite_z) 不是 [0, 1] 均匀分布，而是集中在 0.42~0.58 区间
→ `composite_score = Φ(composite_z) × 100` 实际分数全挤在 **42~58 之间**

**后果：** 修完后顶分依然在 65 附近，**比原来的 72 还低**。然后用 top 5% 分位阈值救场——但 §7.6 输出字段中保留的 `composite_score`（0-100 绝对分）对用户和报表都是误导。

**修订要求（任选其一）：**
- **方案 A（数学修复）**：先归一化方差再做 CDF 映射
  ```
  composite_z_norm = composite_z / sqrt(Σ w_i²)
  composite_score = Φ(composite_z_norm) × 100
  ```
- **方案 B（更稳健）**：放弃正态 CDF，直接用经验分位
  ```
  composite_score = empirical_pct_rank(composite_z) × 100
  ```
  注：方案 B 退化为对 composite_z 做 rank-pct → 与 Defect A 批判的"rank-pct"在最终分数上等价。但因前置已做中性化/正交化/ICIR 加权，仍优于原始 rank-pct。

**建议**：选方案 B，并坦承"最终 0-100 分本质是 composite_z 的横截面分位"，把改进点定位在 **中性化 + 正交化 + ICIR 加权** 三件事上（这才是真正有价值的改动），不要把 CDF 映射包装成"打开尾部"。

---

### B-2 ❌ §7.1 Step 5 "Φ(z) 可拉到 99.5+" 与正态性质冲突（论据错误）

**草案 §2.1 表格 Step 5：**
> 累积分布映射 `score = Φ(z_orthogonal) × 100`，Φ 为标准正态 CDF —— **0~100 分，尾部可拉到 99.5+；不再被 rank 抹平**

**问题：** 如果 z ~ N(0,1)，则 Φ(z) ~ Uniform[0,1]。这恰好等价于 **rank-pct**：
- rank-pct 给第 1 名 0.9998（5000 只里的前 1）
- Φ(z) 给 z=3.5 的标的 0.99977

两者**数学上几乎相同**。所谓"打开尾部"的能力来源于 **不做 CDF 映射，直接用 z 值**（z=4 表示 4 倍标准差，rank-pct=0.9999 看上去差别极小但实际意义巨大）。一旦做了 Φ() 映射，尾部信息就被压回到 [0,1] 区间——与 rank-pct 等价。

**修订要求：** 文档自己批判 rank-pct"抹平尾部"，但提出的方案 Φ(z) 在统计上与 rank-pct 等价，逻辑自洽性破裂。须改：
- 选项 1：删除 Step 5 CDF 映射，最终评分输出 z 值或 z 的分位（与 §7.6 修订 B 一致）
- 选项 2：保留 CDF 映射但明确"目的是输出标准化的 0-100 区间供 UI 显示，不是为了打开尾部信息"，承认 Defect A 的真正修复是 **z 值绝对幅度** 而非 CDF 映射

我推荐选项 2 + 把 Defect A 的诊断重写：rank-pct 真正的问题不是"抹平尾部分数"（那不是问题），而是 **不可跨期可比**（同一 raw 因子值在不同截面下 rank 不同）。这才是 Z-score 标准化的真正价值。

## 2. 重要问题（须澄清 / Major）

### M-1 ⚠️ Defect A 诊断不准确：rank-pct 的真问题是"跨期不可比"，不是"抹平尾部"

业界 A 股量化（华泰金工、广发金工、卖方因子月报）大量使用 rank-pct，因为：
- **稳健**：对异常值天然免疫，不需要 Winsorize 防污染
- **无分布假设**：不假设因子值正态分布
- **业务可解释**：分位即业务排名

rank-pct 的真正缺陷是：
- **跨期不可比**：今日 PE 排名 80% 与昨日 PE 排名 80% 可能对应完全不同的绝对 PE 水平
- **无尺度感**：第 1 名比第 2 名好多少？rank-pct 给等距分位，看不出来

后者在**信号强度排序**上不重要（top 5% 都是候选），在**评分跨期比较**上才致命。

**建议：** 改写 §1.2 Defect A：
> rank-pct 的根本问题是**跨期不可比**——每日重新排名导致同一标的的"评分变化"夹杂了截面分布变化与个股相对位置变化两个信号，无法分离。Z-score 标准化使得跨期评分有限可比（前提是横截面分布近似稳态），这才是 §7.1 升级的核心动机。

### M-2 ⚠️ 正交化引入的副作用须显式承认

**Gram-Schmidt 按 ICIR 顺序正交化** 的后果（草案未说明）：
1. **顺序依赖**：正交化结果完全依赖 ICIR 排序。若两个策略 ICIR 接近（如 0.12 vs 0.11），互换顺序会显著改变 strategy_z_orthogonal 数值
2. **解释性丢失**：正交化后"动量评分"不再是"该股票动量强度"，而是"在剔除趋势策略已解释的部分后，剩下的动量信号"——L1 用户无法理解此语义
3. **稳定性**：每月 rebalance 时 ICIR 排序可能切换 → 正交化顺序切换 → 策略 z 数值跳跃（即使原始因子值未变）

**修订要求：**
- §7.6 输出字段 `score_breakdown_orthogonal` 改名 `score_breakdown_residual` 或加注解"正交化后的残差贡献，已剔除高 ICIR 策略的共线部分"
- 增加约束：当连续两月 ICIR 排序切换时，保留上月顺序（hysteresis），避免月度跳跃
- §2.8 信号解释模板对 L1 用户**不展示正交化后值**，只展示原始 strategy_z 与 ICIR 加权后的贡献

### M-3 ⚠️ Beta 中性化对 A 股个人量化定位存疑

**草案 §2.1 中性化回归：** `residual(x' ~ industry + log(total_mv) + beta)`

A 股研究普遍认为：
- **市值因子**（小市值溢价）：2017 年以前是显著 alpha；2018-2022 大盘风格反向；2024 至今再次小市值占优 → 是否剥离取决于策略意图
- **Beta 因子**（高 Beta 跑赢）：在牛市/反弹期是 alpha，熊市是负 alpha → 与市场状态强相关

对**个人量化决策辅助系统**（SDD 定位）：
- 用户更关心"哪些股票相对其同行业同类有超额"——**行业中性化必要**
- 用户**可能希望保留**市值/Beta 暴露作为策略选择（如"我就要小盘高 Beta"）

**修订建议：**
- Q2 调整推荐选项：**B（仅行业中性化）** 而非 A（行业+市值+Beta）
- 若选 A，须在 §10 RiskChecker 或 §9 信号解释中明确告知用户"已剥离市值/Beta 暴露，候选池可能不包含纯小盘/纯高 Beta 票"
- L3 用户应能选择启用/禁用 Beta 中性化（增加可配置性）

### M-4 ⚠️ ICIR 跨 state 独立计算 + 252 日窗口 + state 数据稀疏的张力

**草案 §2.3：** UPTREND/DOWNTREND/OSCILLATION 各自维护独立 IC 历史。

**问题：** 5y 真机数据 1210 个交易日，按 §3 跨制度验收数据约：
- UPTREND ~400 日
- DOWNTREND ~300 日
- OSCILLATION ~510 日

每个 state 内的 252 日滚动 IC 窗口：
- UPTREND 内只能滚动 ~150 个独立窗口
- DOWNTREND 更紧张：仅 ~50 个独立窗口
- 跨 state 边界时窗口跨越 state，需要剔除还是保留？草案未说明

**修订要求：**
- §2.3 表格 "IC_mean(s, f, state)" 须明确：滚动窗口是否要求**连续 state**（建议要求，否则失去 state 含义）
- 增加 **最小样本量门槛**：state s 在窗口内至少 N 个交易日（建议 N=60），不足则该 state 的 ICIR 沿用上一次有效值或回退至冷启动默认
- 监控 ICIR **置信区间**（基于 sample_size），用于自动下线判断

### M-5 ⚠️ §9.2 策略失效止损改 "ICIR 月度由正转负" 响应过慢

**草案 §2.7：**
> 策略失效止损（保留）：核心贡献策略 **ICIR 在过去 1 个月内由正转负**

**对比原 SDD §10.3：** 核心贡献策略评分**单日**降幅 > 30 分。

两者职能不同：
- **原规则**：捕捉**突发**策略失效（个股层面单日因子崩塌，如黑天鹅事件）
- **新规则**：捕捉**系统性**策略失效（市场风格切换，月度时间尺度）

两者**互补不替代**。月度 ICIR 反应慢，等到 ICIR 由正转负确认时浮亏已超 -8% 触发硬止损。

**修订要求：** 草案 Q9 选项 C "双重触发" 改为推荐：
- 短期触发：核心策略单日 score 降幅 > 30 分（原规则保留）
- 中期触发：核心策略 ICIR 月度由正转负（新增）
- 二者任一触发即出 SELL 信号，触发原因细分写入 reason 字段

## 3. 待商榷点（可保留 / Minor）

### m-1 §1.2 数学上限 "78" 的推断方式

> UPTREND 数学上限约 78（trend=100, mom=100, rev=20, val=100 → 88；但 rev 实际经常 <10 → 又下降）

这是简化推断，建议用蒙特卡洛仿真给出 candidate composite 的实际分布上限（如 P99=72），更有说服力。

### m-2 §2.4 月度 rebalance 与月内 state 切换的相互作用

若月初权重基于上月末 OSCILLATION 计算，但月中切换到 UPTREND，权重不重算？还是即时切换到 UPTREND 的默认/历史权重？草案未说明。建议：**state 切换时即时切换权重**（不等月末），并在月末用新 state 内累积的 IC 更新。

### m-3 §2.6 Top 5% × 5000 = 250 候选的假设

A 股 5000 只标的中：
- 停牌 ~50
- ST/*ST ~150
- 新股次新（涨跌停限制 20%）~100
- 流动性不足（日均成交额 < 5000 万）~1500

实际**可投资宇宙约 3200~3500 只**，top 5% = 160~175 只，与草案"250"相比少 30%。建议改文案为"理论 top 5% 候选约 150~250 只"。

### m-4 §2.8 L1 用户看到 ICIR 数值需 Tooltip

> 驱动权重：趋势 38%（ICIR=0.18）

ICIR 是行业术语，L1 用户难理解。建议：
- L1：只显示权重，不显示 ICIR 数值 → "驱动权重：趋势 38% · 动量 22% · 均值回归 18% · 价值 22%"
- L2/L3：显示 ICIR，配 Tooltip（已在 Phase 10 术语 Tooltip 系统中扩充）

### m-5 §3.1 新表 schema 增强建议

- `factor_ic_history`：增加 `sample_size`（窗口内有效观测数）、`ic_confidence_interval_low/high` 列
- `strategy_weights_history`：增加 `icir_inputs`（JSONB，记录算权重时各策略的 ICIR 输入）便于审计与回滚

### m-6 §3.2 模块变更与"策略可独立回测"的契约

草案承认评分 Step 1~5 上移到 Scorer/Pipeline 层。BacktestEngine 调用单策略回测时如何处理？建议：
- 单策略独立回测 = 跳过 Step 4（正交化）+ Step 5（CDF），输出 strategy_z 直接排序
- 综合回测 = 完整 5 步管线
- 此约定写入 §3.2 + Phase 11 设计文档 + BacktestEngine 接口契约

### m-7 §3.3 BacktestEngine 与 ICIR 的循环依赖

ICIR 计算需要历史 forward returns（已实现的 t→t+20 收益）。**实时信号生成时**，最新 20 个交易日的 ICIR 还没有 forward returns 数据，怎么处理？

建议：ICIR 滚动窗口 lag 20 天（窗口 = [t-272, t-20]，跳过最近 20 个不完整观测），这样 ICIR 在 t 日可用。在文档中明确。

### m-8 §1.3 隐藏缺陷"行业偏差"——示例期错位

草案举例"2024 H2 半导体"——2026-05 视角下属于历史。建议改为"2025 Q1 AI 算力链行情" 或泛化为"行业风格集中期"避免日期错位。

### m-9 §4 Q1 Winsorize 方法选择

A 股量化实操中**百分位截断（1%/99% 或 0.5%/99.5%）**比 MAD 更常见，因为：
- 因子分布常**偏态**（如 PE、ROE），中位数 ± k×MAD 容易把一侧的正常值截掉
- 百分位截断业务上更易解释（"剔除最极端 1%"）

建议 Q1 推荐改为 **B（百分位 1%/99%）**，MAD 作为可选。
