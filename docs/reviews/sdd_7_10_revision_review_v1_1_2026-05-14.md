## 1. P0 残留：B-1.b 方差归一化在 Gram-Schmidt 残差上仍不完整

### 1.1 v1.1 §2.5 公式回顾

```
composite_z_raw(stock) = Σ_i w_i(state) × strategy_z_orthogonal_i(stock)
composite_z(stock)     = composite_z_raw / sqrt(Σ_i w_i²(state))
```

**v1.1 在 §2.5 关键澄清中说：**
> 方差归一化后 composite_z ~ N(0, 1)（**假设 strategy_z_orthogonal 独立同分布且方差 ≈ 1**），Φ(composite_z) ~ Uniform[0, 1]

### 1.2 问题：Gram-Schmidt 残差的方差并不等于 1

Gram-Schmidt 残差化的定义（§2.1 Step 4）：
```
u_1 = z_1
u_2 = z_2 - proj(z_2, u_1) = z_2 - ⟨z_2, u_1⟩/⟨u_1, u_1⟩ × u_1
u_3 = z_3 - proj(z_3, u_1) - proj(z_3, u_2)
...
```

若原始 strategy_z_i 各自标准化后 Var(z_i) = 1，且策略 i 与策略 j 相关系数 ρ_ij ≠ 0，则：
```
Var(u_2) = 1 - ρ_12²  < 1
Var(u_3) = 1 - ρ_13² - residual_correlation_with_u_2²  < 1
...
```

**举例：** 若 trend 与 momentum 截面相关系数 ρ = 0.6（A 股实测常见值）：
- Var(u_trend) = 1
- Var(u_momentum) = 1 - 0.36 = 0.64
- Var(u_reversion) ≈ 0.50（与 trend、momentum 双重残差化）
- Var(u_value) ≈ 0.70

代入默认权重 w = (0.40, 0.25, 0.15, 0.20)：
```
Var(composite_z_raw)
 = 0.16×1.0 + 0.0625×0.64 + 0.0225×0.50 + 0.04×0.70
 = 0.16 + 0.040 + 0.011 + 0.028
 = 0.239

Var(composite_z) = Var(composite_z_raw) / Σ w_i²
                 = 0.239 / 0.285 ≈ **0.838**
```

→ 即归一化后 composite_z 的实际方差 ≈ 0.84，**不是 1**
→ Φ(composite_z) **不是** Uniform[0,1]，而是集中在 ~0.43~0.57 附近
→ composite_score_0_100 实际范围仍然偏窄（同 v1.0 B-1 残留但程度减轻）

### 1.3 影响

- 层 3 `composite_score_0_100` 顶分仍达不到 95+，估计在 70~80 区间
- 层 2 `composite_pct_in_market` 用经验分位，**不受影响**（这是 v1.1 的安全网）
- 但 v1.1 §2.5 "层 3 与层 2 数学上几乎等价" 的论断**不成立**——只有当 composite_z 真的服从 N(0,1) 时才等价

### 1.4 修订要求（B-1.b）

任选其一：

**方案 a（推荐 / 标准做法）**：Step 4 后追加 "重新标准化"
```
Step 4: Gram-Schmidt 残差化 + 残差再标准化
  u_i_orthogonal = (residual_i - mean(residual_i)) / std(residual_i)
```
这样每个 u_i_orthogonal ~ N(0,1)，§2.5 归一化公式才完全成立。
代价：业务上 u_i 的尺度变了（不再是"原始 z 的纯贡献"，而是"残差的标准化"），但归因解读不受影响——仍可对比 strategy_z_raw 与 strategy_z_orthogonal。

**方案 b（替代方案）**：放弃假设独立同分布，归一化用实际方差
```
composite_z = composite_z_raw / sqrt(Var(composite_z_raw, full_market))
```
即用全市场截面实测方差归一化。
代价：每日重新计算横截面方差，但 Phase 11 实现成本几乎为零（一次 std() 调用）。

**方案 c（保守 / 业界更稳健）**：保留 Step 4 原义不变，层 2 经验分位作为唯一信号触发依据，层 3 仅作 UI 显示，**承认 §2.5 层 3 公式只能给出"压缩后的 0-100 分布"**，并在 §2.8 L1 视图加 Tooltip 说明"该分数为相对得分，绝对数值仅供横向参考"

**评审建议**：选**方案 a**（残差再标准化）。这是业界 Barra 流程的标准做法，文档只需在 §2.1 Step 4 加一行"残差化后对每个 u_i 重新做 z-score 标准化"即可。

## 2. P1 校准：§9.2 短期失效止损阈值 3σ 过严

### 2.1 v1.1 §2.7 条件 3

> **策略失效止损 — 短期突发**：核心贡献策略**单日** z 降幅 > **3 个标准差**（对应原 SDD §10.3 "单日 -30 分"，新公式下用 z 表达）

### 2.2 问题：3σ 与原"30 分"刻度严重不匹配

原 SDD §10.3："单日 score 降幅 > 30 分（0-100 制）"。

考虑 score = Φ(z) × 100。score 从 80 降到 50 等于 z 从 0.84 降到 0。z 降幅 = **0.84**，远小于 3σ。

更典型场景：
| 原 score 降幅 | 对应 z 降幅 |
|---|---|
| 从 90 降到 60（高分崩塌） | z 从 1.28 → 0.25，降幅 ≈ 1.0 σ |
| 从 95 降到 50（雷击崩塌） | z 从 1.65 → 0，降幅 ≈ 1.65 σ |
| 从 99 降到 30（极端雷击） | z 从 2.33 → -0.52，降幅 ≈ 2.85 σ |

**3σ 对应的原 SDD 等价 score 降幅约 60~70 分**，远超原 30 分。

### 2.3 后果

- 短期失效止损被实际上"禁用"——A 股 5y 内单日 z 降 3σ 的事件极罕见（< 0.5% 交易日，且多发生在退市边缘票）
- 原 SDD §10.3 设计为"覆盖中等规模因子崩塌"的触发——v1.1 把它收窄到极端事件
- 中期 ICIR 转负触发又是月度的——两个触发之间**留出 1 个月的响应空窗**，反而比原 SDD 响应更慢

### 2.4 修订要求

调整 §2.7 条件 3 阈值至 **1.5σ ~ 2σ**：
- **推荐 1.5σ**：保持与原 30 分降幅同等触发频率（Φ(z) 在 z=1 附近梯度最大，30 分 score 降幅 ≈ 1.5σ z 降幅）
- **可选 2σ**：略严格，捕捉中等以上崩塌
- L3 用户可调

§4 Q9 选项 C 描述同步更新为"短期 z 降幅 > 1.5σ + 中期 ICIR 月度转负"。

## 3. P1 校准：ICIR_WARMUP_DAYS 应为 272 非 252

### 3.1 v1.1 §3.2 / §2.3 配置回顾

§3.2 config：`ICIR_WARMUP_DAYS=252` / `ICIR_LAG_DAYS=20`

§2.3：滚动窗口 `[t-272, t-20]`，IC_daily 使用 `factor_value_{t-20}, return_{t-20→t}`

### 3.2 问题

窗口最早点 τ = t-272 的 IC_daily 需要：
- factor_value at τ = t-272
- return τ → τ+20 = t-272 → t-252

所以最早数据点需要 t-272 的因子值。即**至少需要 272 个交易日的因子历史**才能形成 1 个有效 ICIR 估计。

`ICIR_WARMUP_DAYS=252` 只够覆盖窗口长度，没考虑前置 20 日的因子历史。系统首次部署时 252 日就启用 ICIR，会出现 IC_daily 缺值。

### 3.3 修订要求

将 `ICIR_WARMUP_DAYS` 改为 **272**，并在 §2.3 / §3.2 加注：

> 冷启动判定：可用因子历史 ≥ 272 个交易日（= 滚动窗口 252 + lag 20）才进入稳态期；不足则用冷启动默认权重。

5y 真机数据 1210 日 ≥ 272，所以实际上线时不影响——但首次部署或新增因子时这 20 日差距会咬人。

## 4. P2 优化建议（非阻断）

### 4.1 §2.5 score_change 在 state 切换时的不连续性

v1.1 §2.5 输出表注明 `score_change = composite_z 与前一交易日差`。但 §2.4 提到"state 切换即时换权重"。

→ State 切换日 composite_z 计算用了新权重，与前一日的旧权重 composite_z 不在同一基准上 → score_change 数值会跳跃（与个股因子变化无关）

**建议**：§2.5 输出表加一列 `score_change_attribution`，分解为：
- `delta_from_factor_change`（个股因子值变化贡献）
- `delta_from_weight_change`（权重切换贡献）

或者更简单：state 切换日 score_change 字段填 NULL + Tooltip"权重切换日，跨日变化口径暂不可比"。

### 4.2 §2.8 L2 视图 "MA60 残差 z=+1.8（行业内排名 92%）"

v1.1 §2.8 L2 视图示例：
> MA60 残差 z=+1.8（行业内排名 92%）

"行业内排名 92%" 与"残差 z=+1.8"在 L2 用户眼里语义重复——既然剥离了行业暴露，残差就是"行业内相对位置"，再标"行业内排名 92%"是多此一举。

**建议**：L2 视图改为 "MA60 残差 z=+1.8（已剥离行业 / 市值暴露，表示个股相对同类的超额信号）"，把语义说清而非用百分位重复。

### 4.3 §2.3 IC_daily 命名与 lag 关系不清晰

v1.1 §2.3 表："IC_daily(s, f, t) | 第 t 日，... corr(factor_value_{t-20}, return_{t-20→t})"

"第 t 日的 IC_daily" 但实际计算用的是 t-20 日的因子值和 t-20 到 t 的收益——名为"t 日"实为"以 t-20 日因子为信号源、回测到 t 日为止"。

业界更标准的命名是 `IC_observed_at(t) = IC_for_signal_date(t-20)`，明确"在 t 日观测到、但信号来自 t-20 日"。

**建议**：§2.3 表改为 "IC_observed(s, f, t)" + 一行说明"在 t 日可计算的 IC 估计，使用 t-20 日因子值与 t-20→t 收益"。

## 6. v1.1 新引入的设计要点（评审者认可）

以下是 v1.1 相对 v1.0 的**创新与优化**，评审者表示认可，无修订意见：

1. **三层输出分离（§2.5）**——把"跨期归因 / 信号触发 / UI 显示"三件事用三个不同字段承担，远优于 v1.0 用单一 0-100 分担三责。这是 v1.1 最有价值的设计改进。
2. **Hysteresis 防月度跳跃（Q10 / §2.3）**——连续 2 月排序持续才切换正交化顺序，避免单点波动污染信号。业界做法。
3. **中性化分层默认（Q2）**——行业必做 / 市值&Beta 可选默认关 + L3 可调。比评审建议的"硬选择 B 或 C"更灵活，给用户保留主动暴露权利。
4. **score_breakdown_raw vs score_breakdown_residual 双输出（§2.5）**——业务可解释（raw）与统计严谨（residual）兼顾，L1 用 raw、L2/L3 用 residual。
5. **配置常量声明完整（§3.2）**——11 个新增配置项一次性列出，便于 Phase 11 实施时直接落 `core/config_defaults.py`。
6. **State 切换即时换权重（§2.4）**——比月末等更响应。
