# 第一部分：系统概述

## 1. 产品定义与系统边界

### 1.1 产品定义

QuantPilot 是一款面向个人投资者的**中低频量化交易决策辅助系统**。

### 1.2 系统边界

**系统做什么：**

- 提供数据驱动的买卖建议，附带可解释的策略依据
- 提供科学的仓位与风险管理建议
- 自动记录决策依据，支持系统级复盘与绩效归因
- 通过多渠道推送关键交易信号

**系统不做什么：**

- 不自动下单，不对接券商交易接口（远期可选）
- 不提供实时盘中信号（日线级别，非高频）
- 不承诺收益，不构成投资建议（需合规声明）
- 不处理衍生品、融资融券等杠杆交易

### 1.3 核心价值主张

系统核心解决四个投资决策问题：

| 问题 | 对应能力 | 关键模块 |
|------|----------|----------|
| **选什么**（Alpha 发现） | 多策略因子评分，筛选概率占优标的 | 多策略评分引擎 |
| **何时买卖**（择时） | 市场环境识别 + 信号触发机制 | 市场环境识别 + 信号生成 |
| **买多少**（风险控制） | 科学仓位模型，控制单笔与整体风险 | 投资组合与风险管理 |
| **为何赚/亏**（认知升级） | 系统性绩效归因，形成投资方法论 | 绩效归因与分析 |

---

## 2. 用户层级定义

系统必须**默认适配 L1 用户**，同时开放 L2/L3 的进阶能力。

| 层级 | 用户特征 | 系统适配方式 |
|------|----------|--------------|
| **L1 初级用户** | 无量化背景，时间有限 | 使用全部默认参数，接收简洁的买/卖/持有信号及通俗理由 |
| **L2 进阶用户** | 理解技术指标含义 | 可调整策略参数（如均线周期、阈值），查看详细评分 |
| **L3 专业用户** | 具备量化投资认知 | 可自定义权重映射、编写策略插件、查看因子归因 |

**设计原则：** 所有 L2/L3 功能隐藏在"高级设置"中，不干扰 L1 用户的核心体验。

## 3. 系统总体结构

### 3.1 子系统划分

系统由七大子系统构成：

```
┌─────────────────────────────────────────────────┐
│                  用户交互层                        │
│  ┌──────────┐ ┌──────────┐ ┌──────────────────┐  │
│  │ 提醒与推送 │ │ 用户设置  │ │ 可视化与报告展示  │  │
│  └──────────┘ └──────────┘ └──────────────────┘  │
├─────────────────────────────────────────────────┤
│                  业务逻辑层                        │
│  ┌──────────┐ ┌──────────┐ ┌──────────────────┐  │
│  │ 市场环境   │ │ 多策略    │ │ 信号生成          │  │
│  │ 识别系统   │→│ 评分引擎  │→│ 与股票池管理      │  │
│  └──────────┘ └──────────┘ └──────────────────┘  │
│  ┌──────────┐ ┌──────────┐ ┌──────────────────┐  │
│  │ 投资组合   │ │ 账户与    │ │ 绩效归因          │  │
│  │ 与风控    │ │ 交易管理  │ │ 与分析           │  │
│  └──────────┘ └──────────┘ └──────────────────┘  │
├─────────────────────────────────────────────────┤
│                  数据基础层                        │
│  ┌──────────┐ ┌──────────┐ ┌──────────────────┐  │
│  │ 数据采集   │ │ 数据清洗   │ │ 数据存储与查询    │  │
│  │ 与接入    │ │ 与校验    │ │                  │  │
│  └──────────┘ └──────────┘ └──────────────────┘  │
└─────────────────────────────────────────────────┘
```

### 3.2 每日运行流程

系统核心运行周期为**日级批处理**，于每个交易日收盘后触发：

```
收盘(15:00)
  → ① 数据采集与校验（行情/财务/指数）
    → ② 市场环境识别（判定当日市场状态）
      → ③ 全市场策略评分（遍历标的，逐策略打分）
        → ④ 综合评分与股票池更新（加权合成，刷新关注池）
          → ⑤ 信号生成（对比阈值，产出买卖信号）
            → ⑥ 仓位建议计算（结合账户状态与风控规则）
              → ⑦ 信号推送（通知用户）
                → ⑧ 用户操作（查看信号 / 录入交易 / 忽略）
```

**幂等性要求：** 同一交易日内，重复执行上述流程应产出完全一致的结果。为保障幂等性，批处理开始时必须对输入数据建立**快照（Snapshot）**，全程使用同一数据版本，不受批处理期间数据源推送更新的影响。

### 3.3 模块依赖关系

```
数据基础层 ← 市场环境识别 ← 权重动态调整
数据基础层 ← 多策略评分引擎 ← 综合评分 ← 信号生成
信号生成 + 账户管理 ← 仓位建议
信号生成 ← 提醒推送
账户管理 + 信号历史 ← 绩效归因
```

# 第二部分：数据体系

## 4. 数据需求总览

### 4.1 基础行情数据

| 数据项 | 说明 | 要求 |
|--------|------|------|
| OHLCV | 开盘价、最高价、最低价、收盘价、成交量 | A 股全市场日线 |
| 复权因子 | 累乘式复权因子（以上市首日为基准值1.0），用于按需计算**前复权**或**后复权**序列 | 涵盖除权除息、送转股 |
| 成交额 | 当日成交金额 | 用于流动性过滤 |
| 换手率 | 当日成交量 / 流通股本 | 用于流动性与异动监测 |
| 历史深度 | — | **至少 5 年**（含牛熊完整周期） |

> **复权策略：** 系统存储**原始不复权价格 + 每日复权因子**，不以前复权价格为唯一存储形式。计算引擎根据场景按需派生：
> - **回测引擎**：使用**后复权**序列（以上市首日为基准向前累乘），确保历史价格序列不随新除权事件变化，保证回测结果可重现。
> - **展示层**：动态计算**前复权**序列（以最新价为基准向历史调整），供用户直观阅读。
> - 严禁将动态计算的前复权价格持久化存储为唯一历史数据，以避免违背幂等性原则。

## 5. 量化严谨性要求

以下要求为量化系统的**底线标准**，贯穿数据、回测、信号生成全链路。

### 5.1 Point-In-Time（PIT）原则

> 在 T 日进行计算时，系统只能访问 T 日及之前**已公开**的数据。

具体约束：

- 财务数据以**公告发布日**（而非报告期末日）为可用时点
- 数据修正（如财报更正）以修正公告日为新的可用时点
- 回测引擎必须严格遵守 PIT，**禁止任何形式的未来数据泄露**

### 5.2 幸存者偏差消除

数据集必须包含：

- **已退市股票**的完整历史数据（直至退市日）
- **历史 ST 状态**变化记录
- **历史指数成分股**（各时点的实际成分列表）

回测时的标的池必须基于**彼时的可投资集合**，而非当前的存续股票列表。

### 5.3 不可交易状态处理

| 状态 | 处理规则 |
|------|----------|
| 停牌 | 不可买入/卖出，不参与评分排名 |
| 一字涨停（封死） | 不可买入，可纳入观察但不生成买入信号 |
| 一字跌停（封死） | 不可卖出，需在信号中标注"当前无法成交" |
| 上市首日 | 不参与评分（无历史数据） |

### 5.4 标的过滤（选股宇宙定义）

每日评分前，先从全市场筛出**可投资宇宙**：

| 过滤规则 | 说明 |
|----------|------|
| 排除 ST / \*ST | 风险提示股票 |
| 排除上市不足 60 个交易日 | 次新股缺乏历史数据，评分不可靠 |
| 排除当日停牌 | 不可交易 |
| 排除当日一字涨停 | 无法买入 |
| 排除过去 20 日日均成交额 < 阈值 | 流动性不足（默认阈值：500 万元，可配置） |
| 排除净资产为负 | 最新一期报告期净资产（股东权益）< 0，基本面底线 |
| 排除连续两个报告期净利润亏损且无改善趋势 | 净利润同比增速仍为负值；非 ST 但基本面极差的标的（可配置开关） |
| 排除资产负债率 > 90%（非金融行业） | 极端杠杆风险；金融类行业（银行/证券/保险）豁免此规则 |
| 用户黑名单排除 | 用户手动排除的标的 |
| 用户白名单强制纳入 | 用户手动添加的关注标的（仍需满足停牌过滤） |

### 5.5 数据质量校验

每日数据入库前执行以下校验：

| 校验项 | 规则 | 异常处理 |
|--------|------|----------|
| **完整性** | 当日有行情的标的数量应 ≥ 前一交易日的 95% | 告警，暂停流水线 |
| **时效性** | 数据日期 = 最近交易日 | 拒绝过期数据 |
| **价格有效性** | Low ≤ Open, Close ≤ High | 标记异常，不参与评分 |
| **成交量非负** | Volume ≥ 0 | 标记异常 |
| **复权连续性** | 后复权价格不出现 >20% 的跳变（排除已知除权日） | 告警并人工复核 |

- 不依赖单一数据提供商的特有字段命名或数据格式
- 数据接入层负责将外部数据统一映射为系统内部标准格式（见附录 D）

## 6. 市场环境识别系统

### 6.1 功能目标

动态感知大盘环境，为策略权重调整和仓位管理提供宏观依据。

### 6.2 状态分类

| 状态 | 含义 | 典型特征 |
|------|------|----------|
| **上涨趋势** | 市场整体处于中期上升通道 | 均线多头排列，趋势指标确认 |
| **下跌趋势** | 市场整体处于中期下降通道 | 均线空头排列，趋势指标确认 |
| **震荡市** | 无明确方向，区间波动 | 均线交织，趋势指标信号弱 |

### 6.3 判定指标与逻辑

以**沪深 300 指数**为主要参考（可配置）：

| 指标 | 计算方式 | 判定规则 |
|------|----------|----------|
| 均线排列 | MA20 与 MA60 的相对位置 | MA20 > MA60 → 多头；MA20 < MA60 → 空头 |
| ADX | 14 日平均方向指数 | ADX > 25 → 趋势明确；ADX ≤ 25 → 震荡 |
| 布林带宽度 | (上轨 - 下轨) / 中轨 | 辅助判断波动率环境 |
| 指数位置 | 收盘价相对 MA60 的偏离度 | 辅助确认趋势方向 |

**综合判定逻辑：**

```
如果 ADX > 25（趋势明确）：
    如果 MA20 > MA60 且 收盘价 > MA20 → 上涨趋势
    如果 MA20 < MA60 且 收盘价 < MA20 → 下跌趋势
    否则 → 震荡市
如果 ADX ≤ 25 → 震荡市
```

### 6.4 输出规范

每日输出：

| 字段 | 类型 | 说明 |
|------|------|------|
| date | 日期 | 交易日 |
| market_state | 枚举 | UPTREND / DOWNTREND / OSCILLATION |
| trend_strength | 0-100 | 基于 ADX 归一化的趋势强度 |
| state_changed | 布尔 | 相比前一交易日是否发生状态转换 |
| description | 文本 | 状态说明文字（面向 L1 用户） |

### 6.5 状态转换规则

- **防抖动机制：** 状态切换需连续 **3 个交易日**满足新状态条件方可确认转换，避免因单日波动频繁切换
- **状态变化事件：** 状态转换时记录日志，并可触发通知
- **历史记录：** 保存完整的每日市场状态序列，用于回测与归因

## 7. 多策略评分引擎

### 7.1 策略模块化设计规范

每个策略必须满足以下接口契约：

| 契约项 | 要求 |
|--------|------|
| **独立输入** | 仅依赖数据层提供的标准化数据，策略间无数据依赖 |
| **独立评分** | 输出经 §7.1 评分管线处理后的 z 值、分位与 0–100 显示分 |
| **解释输出** | 附带可读的评分理由文本（面向 L1 用户可理解） |
| **可单独回测** | 可独立运行历史回测，验证策略有效性（回测接口规范见 7.7 节） |
| **参数可配置** | 核心参数（如周期、阈值）可通过配置调整 |
| **幂等性** | 相同输入产出相同结果 |

**评分管线（5 步规范）：** 每个原始因子值 → 单策略评分 → 综合评分均经过以下管线（参考 Barra / Axioma / 卖方金工标准流程）：

| 步骤 | 操作 | 数学定义 | 目的 |
|---|---|---|---|
| **1. 去极值** | Winsorize（百分位 1%/99%） | `x' = clip(x, percentile_1%, percentile_99%)` | 排除单只票极端值对中性化回归的污染（A 股因子偏态多，百分位优于 MAD） |
| **2. 中性化** | 横截面回归取残差 | `x'' = residual(x' ~ industry_dummies + log(total_mv) [+ beta])`，行业**强制开**、市值**默认开**、Beta**默认关**（L3 可调） | 剥离行业风格、剥离市值风格（避免用户被动押注市值因子，A 股市值溢价时变性强）；保留 Beta 主动暴露权利 |
| **3. 标准化** | Z-score | `z = (x'' - mean) / std` | 跨期有限可比；保留尾部信号绝对幅度 |
| **4a. 正交化** | Gram-Schmidt 残差化 | 按策略级 ICIR 排序，逐策略剔除前序策略的线性投影 `u_i = z_i - Σ_{j<i} proj(z_i, u_j)`；Hysteresis 防月度跳跃（连续 2 月排序持续才切换） | 消除策略间共线性，4 个 strategy_factor 互相独立 |
| **4b. 残差再标准化** | Z-score | `u_i_normalized = (u_i - mean(u_i)) / std(u_i)` | Gram-Schmidt 残差化后 `Var(u_i) = 1 - Σ_{j<i} ρ_ij² < 1`，必须重新标准化使 `u_i ~ N(0, 1)`，作为 §7.6 综合评分方差归一化公式的数学前提（业界 Barra 流程标准做法） |
| **5. 输出分层** | 三层输出 | `strategy_z`（归因/跨期对比）+ `strategy_pct_in_market`（信号触发）+ `strategy_score_0_100 = Φ(z) × 100`（仅 UI 显示） | 把跨期归因 / 信号触发 / 用户呈现三件事分开，避免单一输出承担多职 |

**重要约束：**

- **不宣称"CDF 映射打开尾部"**——Φ(z) 与 rank-pct 数学等价，尾部尺度感来源于 Step 3 Z-score 的绝对幅度，不是 Step 5 CDF
- `strategy_score_0_100` **仅供 UI 显示**，业务决策（信号触发 / 归因 / 排序）一律用 z 或 pct，禁止用 0–100 绝对值
- **单策略独立回测**：BacktestEngine 单策略模式跳过 Step 4 + Step 5，直接对 strategy_z 横截面排序选股（与"单策略可独立验证"契约一致）

> **跨期可比性：** Z-score 保留因子值的绝对幅度（z=3.5 vs z=4.5 是真实 1 个标准差差距），用于归因和跨期对比；横截面分位（pct）解决信号触发阈值跨期失效问题。两者**互补**——z 用于跨期对比，分位用于信号触发，0–100 仅供 UI 易读展示。

> **历史数据兼容：** 旧 rank-pct 算法的历史评分快照（Phase 11 上线之前）继续保留在 `candidate_pool` / `signal_score_snapshot` 表中作为 baseline，新评分独立写入新列（详见 §15.2 / Phase 11 §11-1）。

### 7.2 内置策略

> **表注：** §7.2.1~§7.2.4 各策略内"因子权重"为**冷启动默认值**：
> - **冷启动期**（系统首次部署 ≤ 252 个交易日 IC 历史，或任一 state 子集 < 60 日）：使用表内固定权重
> - **稳态期**：策略内因子权重由 §7.4 ICIR 滚动校准动态生成（业界标准做法）
> - 用户可在 Settings L3 模式下查看当前生效的实时权重；IC 数据不足时自动回退冷启动默认

#### 7.2.1 趋势策略

**投资逻辑：** 识别处于中期上升趋势的标的，顺势而为。

| 因子 | 计算逻辑 | 权重（策略内） |
|------|----------|----------------|
| 均线多头排列 | MA5 > MA10 > MA20 > MA60 的满足程度 | 40% |
| MACD 状态 | DIF 与 DEA 的相对位置及变化趋势 | 30% |
| 价格突破 | 收盘价 vs 近 N 日（默认20日）最高价 | 30% |

**理由模板：** "均线{多头/空头}排列，MACD {金叉/死叉}，价格{突破/未突破}近期高点。"

> **共线性研究：** MA 多头排列因子与价格突破因子在 5 年截面上 Spearman 相关 > 0.8（高度共线）。§7.1 Step 4 因子正交化自动处理此共线性；不要求重构因子定义，但在 ICIR 校准下原 30% 突破权重可能被显著压缩。

#### 7.2.2 均值回归策略

**投资逻辑：** 识别短期超跌标的，捕捉向均值回归的反弹机会。

> **重要约束：** 均值回归策略在**下跌趋势**市场状态下权重显著降低（见 7.5 节），以避免在系统性下跌中接刀持续下跌的标的。该策略在下跌趋势中主要作为辅助参考，而非主驱动因素。

| 因子 | 计算逻辑 | 权重（策略内） |
|------|----------|----------------|
| RSI 超卖 | 14 日 RSI 值，越低得分越高 | 35% |
| 乖离率 | 收盘价偏离 20 日均线的百分比 | 35% |
| 布林带位置 | 收盘价在布林带中的相对位置 | 30% |

**理由模板：** "RSI={值}处于{超卖/中性/超买}区域，股价偏离均线{X}%，位于布林带{上/中/下}轨附近。"

#### 7.2.3 动量策略

**投资逻辑：** 过去一段时间表现强势的标的，短期内延续强势的概率较大。

| 因子 | 计算逻辑 | 权重（策略内） |
|------|----------|----------------|
| 3 个月涨幅 | 近 60 个交易日的收益率排名 | 40% |
| 6 个月相对强度 | 近 120 个交易日的收益率排名 | 35% |
| 行业相对强度 | 标的涨幅 vs 所属行业平均涨幅 | 25% |

**理由模板：** "近3个月涨幅排名前{X}%，近6个月相对强度{强/中/弱}，行业内表现{领先/居中/落后}。"

> **动量策略注意事项：**
> - 需剔除近一个月涨幅排名前 5% 的极端标的（短期反转效应），可配置。
> - 建议采用**风险调整动量**（涨幅 / 历史波动率）作为增强选项（L2+ 可配置），减少对高波动标的的偏向，识别"稳定上涨"而非"大起大落"的标的。

#### 7.2.4 价值策略

**投资逻辑：** 识别估值处于历史低位、基本面质量尚可的标的。

| 因子 | 计算逻辑 | 权重（策略内） |
|------|----------|----------------|
| PE 历史分位 | 当前 PE(TTM) 在过去 **5 年**中的百分位（越低得分越高） | 35% |
| PB 历史分位 | 当前 PB 在过去 **5 年**中的百分位（越低得分越高） | 30% |
| ROE 质量 | 最近一期 ROE 的横截面排名 | 35% |

**理由模板：** "PE 处于近5年{X}%分位，PB 处于{X}%分位，ROE 排名前{X}%。"

> **价值陷阱规避：** PE/PB 低不代表有价值，需结合 ROE 质量因子过滤——ROE 低于行业中位数的标的，价值策略得分上限为 50 分。

### 7.4 因子 ICIR 监控与权重校准

**核心约束（lag 20 日）：** ICIR 计算依赖**已实现**的前向收益（t→t+20 日），因此 t 日实时可用的 ICIR 实际是 **`[t-272, t-20]` 窗口**（lag 20 个交易日跳过未完成观测）。Phase 11 实施时必须严格遵守此 lag。

**冷启动判定：** 因子可用历史 ≥ **272 个交易日**（= 252 滚动窗口 + 20 lag）才进入稳态期；不足则回退冷启动默认权重。`ICIR_WARMUP_DAYS=272` 是首次部署或新增因子时的硬性门槛。

对每个策略 × 每个因子 × 每个市场状态，维护以下指标：

| 指标 | 定义 | 滚动窗口 | 更新频率 |
|---|---|---|---|
| **IC_daily(s, f, t)** | 在第 t 日**观测到**的 IC 估计，使用 t-20 日因子值与 t-20→t 的已实现收益：`corr(factor_value_{t-20}, return_{t-20→t})`（不区分 state）。**命名 lag 含义**：IC_daily(t) 名为"t 日的 IC"，但信号源是 t-20 日——体现"因子值 → 收益验证"的 20 日 lag 关系（业界惯例保留 IC_daily 命名）。 | 单点 | 每日 |
| **IC_mean_window(s, f)** | 在 `[t-272, t-20]` 窗口内 IC 均值 | 全窗口 ~252 日 | 每月最后交易日 |
| **IC_mean_state(s, f, state)** | 同窗口内按 state 子集（state_{t-20}==state 的子集）计算的 IC 均值 | 子集大小 ≥ 60（最小样本门槛） | 每月 |
| **IC_std_state(s, f, state)** | 同子集的 IC 标准差 | 子集大小 ≥ 60 | 每月 |
| **ICIR(s, f, state)** | IC_mean_state / IC_std_state | 子集大小 ≥ 60 | 每月 |
| **sample_size(s, f, state)** | 子集内有效观测数 | — | 每月 |
| **ic_ci_low / ic_ci_high(s, f, state)** | IC 95% 置信区间 | bootstrap 1000 次 | 每月 |
| **t-stat(s, f, state)** | ICIR × sqrt(sample_size) | — | 每月 |
| **半衰期(s, f)** | IC 自峰值衰减至一半所需的日数 | 滚动 504 日 | 每月 |

**IC 计算规范：**
- 使用 **Spearman 秩相关系数**（Rank IC）
- 下期收益窗口：默认 **20 个交易日**（约一个月），可配置（支持 5/10/20/60 日）
- 计算时剔除涨跌停和停牌导致的异常收益

**样本量不足的处理：**

```
若 sample_size(s, f, state) < 60:
    该 (state, factor) 的 ICIR 沿用上次有效值
    若无上次有效值 → 回退冷启动默认权重
    连续 3 个月样本不足 → 告警"该 state 数据稀疏"
```

**跨 state 边界处理：** 主滚动窗口 `[t-272, t-20]` **不要求 state 连续**（A 股 state 切换频繁，强制连续在 5y 内基本不可达成）。在此 252 日窗口内按 state_{t-20} 子集分别计算各 state 的 IC——观察日的 state 决定该观测归属哪个子集。

**ICIR 加权机制（V1.0 行为，Phase 11 实施后）：**

1. **策略内因子权重**：在策略 s 内，因子 f 的权重 = `max(ICIR(s, f, state), 0) / Σ_j max(ICIR(s, j, state), 0)`
2. **策略级权重**（§7.5 由此衍生）：策略 s 的权重 = `max(策略级 ICIR(s, state), 0) / Σ_k max(策略级 ICIR(k, state), 0)`
3. **冷启动 fallback**（三条件 OR）：
   - 历史 < 272 个交易日 → 使用默认权重
   - 或所有 ICIR ≤ 0 → 使用默认权重
   - 或任一 state 子集 < 60 日 → 该 state 使用默认权重，其他 state 用 ICIR

**因子自动下线规则：**

| 触发条件 | 处置 |
|---|---|
| ICIR < 0 持续 6 个月 | 该 (state, factor) 权重强制置 0；推送告警 |
| t-stat < 1.96（不显著）持续 12 个月 | 权重置 0；通知考虑替换因子定义 |
| 半衰期 < 5 日 | 权重减半；通知"该因子可能为短期噪声" |
| sample_size < 60 连续 3 月 | 告警"该 state 数据稀疏"（不下线） |

**Hysteresis 防月度跳跃：** 策略级权重 rebalance 时，若 ICIR 排序较上月变化 → 仅在**连续 2 个月排序保持新顺序**后才切换正交化顺序。短期 1 个月单点排序切换被忽略，避免数值跳跃污染信号（L3 用户可关闭，用于研究模式）。

### 7.5 策略级权重机制

综合评分中各策略的权重由 §7.4 ICIR 校准产出，三个市场状态各自独立维护。

**冷启动默认矩阵**（系统首次部署或 IC 历史不足 272 日时使用）：

| 市场状态 | 趋势 | 动量 | 均值回归 | 价值 |
|---|---|---|---|---|
| **UPTREND** | 40% | 25% | 15% | 20% |
| **DOWNTREND** | 10% | 5% | 15% | 70% |
| **OSCILLATION** | 15% | 15% | 40% | 30% |

**权重设计逻辑：**

- 上涨趋势：顺势策略（趋势+动量）主导，跟随市场方向
- 下跌趋势：价值策略高度主导（70%），提供安全边际；均值回归权重压缩至 15%（避免在系统性下跌中持续接刀），趋势/动量各保留小比例以监控个股异动
- 震荡市：均值回归最有效，价值提供安全垫

**稳态期生效权重**（IC 历史 ≥ 272 日且各 state 子集 ≥ 60）：
```
w_strategy(state) = max(ICIR_strategy(state), 0) / Σ_k max(ICIR_k(state), 0)
```

**State 切换即时换权重：** 若 MarketStateEngine 判定 state 切换（如 UPTREND → OSCILLATION），权重**即时切换**到新 state 的当前 ICIR 加权（不等月末）。月末 rebalance 用窗口内新 state 子集累积 IC 重算。

**可配置性：**

- **L1 用户**：无配置选项，系统按 ICIR 自动加权
- **L2 用户**：可在"自动 ICIR"与"固定默认矩阵"之间切换
- **L3 用户**：可手工覆盖具体 state 的权重；可配置市值/Beta 中性化是否启用（详见 §7.1 Step 2）；可关闭 Hysteresis（用于测试 / 研究）；查看 `weights_source` 与 ICIR 历史

**月度 rebalance：** 每月最后一个交易日收盘后计算新权重，次月生效；rebalance 当日的评分基于旧权重保持平滑过渡（state 切换日除外，按"即时换权重"规则处理）。

### 7.6 综合评分

**三层输出公式：**

```
# 层 1：综合 z（含方差归一化，跨期可比）
composite_z_raw(stock) = Σ_i w_i(state) × strategy_z_orthogonal_i(stock)
composite_z(stock) = composite_z_raw / sqrt(Σ_i w_i²(state))

# 层 2：综合分位（用于信号触发，独立于 z 数值）
composite_pct_in_market(stock) = empirical_rank(composite_z, full_market) ∈ [0, 1]

# 层 3：综合 0~100 分（仅供 UI 显示，禁止用于业务决策）
composite_score_0_100(stock) = Φ(composite_z) × 100
```

**方差归一化的数学前提：** strategy_z_orthogonal 经 §7.1 Step 4b "残差再标准化" 后服从 N(0, 1) 且 4 个策略残差互相独立——此时 `Var(composite_z) = Σ wᵢ² / Σ wᵢ² = 1`，composite_z ~ N(0, 1)，Φ(composite_z) ~ Uniform[0, 1]。若省略 Step 4b，则 `Var(u_i) < 1`（如 trend-momentum ρ=0.6 时 Var(u_momentum)=0.64），导致 Var(composite_z) ≈ 0.84、Φ(composite_z) 集中在 0.43~0.57，顶分塌缩到 70~80 —— 这是必须执行 Step 4b 的根本原因。

**关键澄清：**

- 层 3 的 `composite_score_0_100` 与层 2 的 `composite_pct_in_market` **数学上几乎等价**（仅小数点尾差异）——故 0–100 分**严禁**用于触发信号，触发一律用 `composite_pct`
- 业务三件事**严格分开**：
  - **跨期归因**：`composite_z`（保留尺度感与正负号）
  - **信号触发**：`composite_pct_in_market`（top 5% 等绝对分位，跨期稳定）
  - **UI 显示**：`composite_score_0_100`（用户易读的 0–100 分数）

**正交化语义注意：**

- `strategy_z_orthogonal` 是 Gram-Schmidt 残差化后的"残差贡献"，**不等于"该策略对该股票的强度"**
- 例：trend 排第一时，momentum 的 z_orthogonal 是"剔除 trend 已解释部分后的剩余动量信号"
- 归因输出需同时保留 `strategy_z_raw`（正交化前，业务可解释）与 `strategy_z_orthogonal`（正交化后，参与综合评分加权）

**输出字段：**

| 字段 | 说明 |
|------|------|
| stock_code | 股票代码 |
| composite_score | 综合得分（0–100），`Φ(composite_z) × 100`（仅 UI 显示） |
| composite_z | 综合 Z-score（跨期可比基线，归因主用） |
| composite_pct_in_market | 当日全市场分位（0.0~1.0，信号触发主用） |
| score_breakdown_raw | 各策略 z_raw / 权重 / 贡献（**业务可解释**，L1 用户展示） |
| score_breakdown_residual | 正交化后各策略 z_orthogonal / 贡献（**L2/L3 用户展示**，归因审计用，**L1 隐藏**） |
| score_change | 与前一交易日的 composite_z 变化（不再用 composite_score）。**State 切换日填 NULL**——§7.5 "state 切换即时换权重" 使切换日 composite_z 与前一日基准不可比，前端展示用 Tooltip 标注"权重切换日，跨日变化暂不可比" |
| explanation | 综合理由文本（合并各策略 reason，**L1 不含 ICIR/正交化术语**，见 §9.3 分层呈现） |
| weights_source | "icir" / "default_matrix" / "user_override" |
| hysteresis_status | "stable" / "pending_switch"（透明化排序切换状态） |

## 9. 信号生成系统

### 9.1 买入信号

**触发条件（全部满足）：**

1. `composite_pct_in_market ≤ buy_pct_threshold`（默认 5%，即全市场 top 5%）
2. 标的处于可投资宇宙中（非停牌、非涨停封死、非 ST）
3. 当前无该标的持仓，或符合加仓规则（见 10.1 节）
4. 过去 20 日日均成交额达到流动性阈值

**信号强度（分位定义）：**

| 强度 | 分位条件 |
|---|---|
| **STRONG** | `composite_pct_in_market ≤ 1%`（约前 30~35 只） |
| **MODERATE** | `1% < composite_pct_in_market ≤ 5%`（约前 30~175 只） |

**信号数量预期：**

- A 股 ~5500 只标的，**可投资宇宙约 3200~3500 只**（剔除停牌 ~50 / ST 类 ~150 / 新股次新 ~100 / 流动性不足 ~1500~2000）
- 理论 top 5% 候选 = 160~175 只
- 经持仓状态过滤 → 实际 BUY 信号 5~30 条/日
- 经 §10 RiskChecker BLOCK（行业集中度等）→ **建议交付 5~15 条/日**

**信号输出内容：**

| 字段 | 说明 | 示例 |
|------|------|------|
| stock_code | 股票代码 | 600519 |
| stock_name | 股票名称 | 贵州茅台 |
| signal_type | 信号类型 | BUY |
| composite_score | 综合得分（0–100，UI 显示） | 87 |
| composite_z | 综合 Z-score（跨期可比） | 2.41 |
| composite_pct_in_market | 当日全市场分位（信号触发依据） | 0.008（top 0.8%） |
| signal_strength | 信号强度 | STRONG / MODERATE |
| weights_source | 权重来源 | icir / default_matrix / user_override |
| top_contributors | 主要贡献策略（前 2，按 score_breakdown_raw） | "趋势策略(z=+2.1, 权重 38%), 价值策略(z=+1.6, 权重 22%)" |
| explanation | 综合理由（L1 简洁版，详见 §9.3） | "全市场 top 0.6%（强买入），技术面与估值均显著优于行业同类。" |
| suggested_price | 建议买入价区间 | 以信号生成日收盘价为基准的 [收盘价×0.99, 收盘价×1.02]；若次日开盘价超出区间上限，信号不建议追高，应等待回调或忽略本次信号 |
| suggested_position | 建议买入金额 | 总资产的 10%（即 10,000 元） |
| stop_loss_price | 参考止损价 | 1652 元（较建议买入价 -8%） |
| liquidity_note | 流动性提示 | "近20日日均成交额2.3亿元，流动性充足" |
| t1_warning | T+1 风险提示 | "A股T+1制度：买入当日不可卖出" |

### 9.2 卖出信号

**触发条件（任一满足）：**

1. **评分跌出**：持仓股 `composite_pct_in_market ≥ sell_pct_threshold`（默认 70%，即跌入全市场 bottom 30%）
2. **硬止损**（保留）：持仓浮亏达到止损比例（默认 -8%）
3. **策略失效止损 — 短期突发**：核心贡献策略**单日** z 降幅 > **1.5 个标准差**（默认 `SHORT_TERM_FAILURE_SIGMA=1.5`，L3 可调 0.5σ~3σ）
4. **策略失效止损 — 中期系统性**：核心贡献策略 ICIR 在过去 1 个月内由正转负（系统性风格切换）

**条件 3 刻度校准推导**（与历史 SDD §10.3 "单日 score 降 30 分"等价频率）：原 "单日降 30 分" 在 `score = Φ(z) × 100` 下对应：

- 起点 z=1.28 (score=90) → 终点 z=0.25 (score=60)：z 降幅 ≈ **1.0σ**
- 起点 z=1.65 (score=95) → 终点 z=0.38 (score=65)：z 降幅 ≈ **1.27σ**
- 起点 z=2.33 (score=99) → 终点 z=0.50 (score=69)：z 降幅 ≈ **1.83σ**

选 **1.5σ** 居中，5y A 股回测约 0.5%~1.5% 交易日触发——既不过敏感也不过迟钝。

**条件 3 与 4 互补不替代：**

- 短期触发（条件 3）捕捉个股黑天鹅 / 财报雷 / 单日因子崩塌
- 中期触发（条件 4）捕捉行业风格切换 / 因子失效（如 2018-2022 价值 → 成长切换）
- 二者任一触发即出 SELL 信号，`trigger_reason` 字段细分明细

**持仓中性区：** 持仓股 `composite_pct_in_market` 落在 5%~70% 之间不产生信号（持有状态）——分位机制天然规避绝对评分的跨期失效。

**信号输出内容：**

| 字段 | 说明 |
|------|------|
| stock_code / stock_name | 标的信息 |
| signal_type | SELL |
| composite_score | 当前综合得分（0–100，UI 显示） |
| composite_z | 当前综合 Z-score |
| composite_pct_in_market | 当日全市场分位 |
| weights_source | 权重来源（icir / default_matrix / user_override） |
| trigger_reason | 触发原因细分（pct_above_sell / hard_stop_loss / short_term_z_drop / mid_term_icir_flip） |
| explanation | 详细理由（按 §9.3 分层呈现） |
| suggested_price | 建议卖出价区间 |
| holding_pnl | 当前持仓盈亏 |
| holding_days | 持有天数 |

### 9.3 信号解释要求

所有信号**必须可解释**。解释**按用户层级分层**呈现（L1 隐藏 ICIR / 正交化技术细节，L2/L3 按需展开）：

#### L1 用户视图（默认）

简洁、避免技术术语。基于 `score_breakdown_raw`（正交化前，业务可解释）：

- **得分构成**："综合评分 87 分（全市场 top 0.6%），驱动权重：趋势 38% · 动量 22% · 均值回归 18% · 价值 22%"
- **核心驱动因子**："主要驱动：均线多头排列 · MACD 金叉 · PE 处于近 5 年 18% 分位"
- **L1 摘要**："该股票位列全市场前 1%（强买入信号），技术面与估值均显著优于行业同类。"

**L1 隐藏内容：** ICIR 具体数值 · 正交化后的 `score_breakdown_residual` · Hysteresis 状态 · 中性化前/后值。

#### L2 用户视图（在 L1 基础上追加）

- **权重来源透明化**："驱动权重：趋势 38%（ICIR=0.18，自动加权）· 动量 22%（ICIR=0.10）· ..."
- **数据血缘**："因子值（中性化后）：MA60 残差 z=+1.8（已剥离行业 / 市值暴露，表示个股相对同类的超额信号）· PE 残差 z=-1.6（估值显著优于行业内同类）"

#### L3 用户视图（在 L2 基础上追加）

- **正交化贡献**："正交化分解：趋势纯贡献 z=+1.4，动量去除趋势残差后 z=+0.8（即原始动量信号中 30% 已被趋势解释）"
- **审计字段**："weights_source: icir / hysteresis_status: stable / 计算日 IC 样本量：UPTREND=180 / DOWNTREND=72 / OSCILLATION=240"

### 9.4 信号生命周期

| 状态 | 说明 |
|------|------|
| **NEW** | 新产生，尚未被用户查看 |
| **VIEWED** | 用户已查看 |
| **ACTED** | 用户已据此录入交易 |
| **EXPIRED** | 信号产生后超过 N 个交易日（默认 3 日）未操作，自动过期 |
| **SUPERSEDED** | 被同一标的的更新信号取代 |

系统保留**全部历史信号记录**，用于绩效归因和回测验证。

### 9.5 信号冲突处理

- 同一标的同日不会同时产生买入和卖出信号（卖出信号仅针对已持仓标的）
- 若持仓标的的评分在买入阈值与卖出阈值之间，不产生任何信号（持有状态）
- 多只标的同时触发买入信号时，按综合评分从高到低排序展示，仓位建议根据可用资金动态分配

## 10. 投资组合与风险管理系统

### 10.1 仓位控制模型

#### 第一阶段：固定比例法（V1.0，默认模式）

| 参数 | 默认值 | 可配置范围 | 说明 |
|------|--------|------------|------|
| 单笔仓位比例 | 10% | 5%-20% | 单次买入占总资产的比例 |
| 单股持仓上限 | 20% | 10%-30% | 单只标的占总资产的最大比例 |
| 总仓位上限 | 80% | 50%-100% | 股票持仓合计占总资产的最大比例 |
| 最低现金保留 | 20% | 0%-50% | 始终保留的现金比例 |

**加仓规则：**

已持仓标的再次满足买入条件（评分 > 买入阈值）时，满足以下**任一**条件可提示加仓：

- 当前持仓盈利（持仓浮盈 > 0）
- 当前价格与首次建仓成本价的偏离在 ±10% 以内（可配置），且市场状态为上涨趋势或震荡市

加仓后该标的总仓位不得超过单股持仓上限；每次加仓金额 = 单笔仓位比例 × 总资产。

**市场状态调节：**

| 市场状态 | 总仓位上限调节 |
|----------|----------------|
| 上涨趋势 | 维持设定值（如 80%） |
| 震荡市 | 降至设定值的 75%（如 60%） |
| 下跌趋势 | 降至设定值的 50%（如 40%） |

#### 第二阶段：分数凯利法（V1.5，进阶模式）

基于历史胜率与盈亏比计算理论最优仓位：

```
K% = W - (1-W) / R

W = 该策略组合的历史胜率
R = 平均盈利 / 平均亏损（盈亏比）
```

- 默认采用 **0.25-0.5 倍凯利**（分数凯利），平衡增长与波动
- 凯利比例的选择因子：信号强度越高，越接近 0.5 倍凯利

#### 第三阶段：波动率目标化（V2.0，专业模式）

- 根据标的 ATR（平均真实波幅）动态调整头寸
- 目标：维持组合整体年化波动率在设定范围内（如 15%-20%）
- 高波动标的自动减仓，低波动标的可适度增仓

### 10.2 风险矩阵

| 监控维度 | 规则 | 阶段 |
|----------|------|------|
| **单股集中度** | 任一标的持仓 ≤ 单股上限 | V1.0 |
| **行业集中度** | 同一申万一级行业持仓合计 ≤ 30% | V1.0 |
| **市值风格** | 监控大/中/小盘持仓占比，提示偏离 | V1.5 |
| **因子暴露** | 监控组合在 Value/Momentum/Volatility 因子上的总暴露 | V2.0 |
| **相关性** | 持仓标的间相关系数矩阵，提示高相关聚集 | V2.0 |

**风险突破处理：** 当持仓超出限制时（通常因股价变动导致），系统标记告警并建议减仓，但不强制操作。

### 10.3 止损机制

| 类型 | 规则 | 说明 |
|------|------|------|
| **硬止损** | 持仓浮亏 ≥ X%（默认 8%） | 无条件触发卖出信号 |
| **移动止损** | 从持仓最高盈利回撤 ≥ Y%（默认 15%） | 保护已有利润（V1.5） |
| **时间止损** | 持有 N 个交易日（默认 30 日）后仍无盈利 | 提示关注，非强制（V1.5） |
| **策略失效止损** | 核心贡献策略评分骤降（单日降幅 > 30 分） | 触发卖出信号 |

### 10.4 压力测试（V2.0）

支持模拟极端场景下的组合表现：

| 场景 | 模拟方式 |
|------|----------|
| 2008 年全球金融危机 | 应用历史指数跌幅到当前持仓 |
| 2015 年 A 股流动性危机 | 应用历史指数跌幅 + 考虑跌停卖不出 |
| 自定义：指数下跌 X% | 用户输入跌幅，按 Beta 估算个股影响 |

输出：预期组合最大回撤、各持仓预计亏损。

### 10.5 交易成本模型

信号建议中必须考虑交易成本：

| 费用项 | 默认值 | 说明 |
|--------|--------|------|
| 佣金 | 0.025%（双向） | 可配置 |
| 印花税 | 0.05%（仅卖出） | 按政策调整 |
| 滑点估算 | 0.1% | 基于流动性动态调整（V1.5） |

## 12. 绩效归因与分析

### 12.1 基础绩效指标

| 指标 | 计算方式 | 展示周期 |
|------|----------|----------|
| 累计收益率 | (当前总资产 - 初始资金) / 初始资金 | 实时 |
| 年化收益率 | 累计收益率按自然日折算 | 实时 |
| 最大回撤 | max(1 - 当前净值 / 历史最高净值) | 实时 |
| 夏普比率 | (年化收益 - 无风险利率) / 年化波动率；**无风险利率采用中国10年期国债收益率（季度更新，系统自动获取或可手动配置），默认值3%** | 月度 |
| 胜率 | 盈利交易笔数 / 总交易笔数 | 实时 |
| 盈亏比 | 平均盈利金额 / 平均亏损金额 | 实时 |
| 基准对比 | 同期沪深300收益率 | 实时 |

### 12.2 收益归因

**个股维度：**

- 每只已平仓标的的盈亏金额、盈亏率、持有天数、买卖信号类型
- 按盈亏排序，展示最佳/最差交易

**行业维度：**

- 各行业的合计盈亏贡献
- 行业配置偏离基准的超额收益（V1.5+）

**策略维度：**

- 按触发信号的主要策略分组，统计该策略驱动的交易的累计收益、胜率、盈亏比
- 识别哪些策略在近期贡献正收益，哪些拖累

### 12.3 因子归因（V1.5+）

使用多因子回归分析拆解组合收益来源：

```
Rp = α + β_market × Rm + β_size × Size + β_value × Value + β_momentum × Momentum + ε
```

输出示例："本月收益中，60% 来自行业配置超额，25% 来自动量因子暴露，Alpha 贡献 15%。"

### 12.4 行为分析

监测用户交易行为，辅助改善投资纪律：

| 指标 | 说明 |
|------|------|
| 平均持仓周期 | 从买入到卖出的平均交易日天数 |
| 交易频率 | 月均交易笔数 |
| 信号遵守率 | 用户实际交易中有多少比例是跟随系统信号的 |
| 止损执行率 | 触发止损信号后，用户实际执行卖出的比例 |
| 追涨行为 | 在信号过期后才买入的比例（可能追高） |
| 盈亏分布 | 直方图展示单笔交易盈亏的分布形态 |

### 12.5 报告输出

| 报告类型 | 频率 | 内容 |
|----------|------|------|
| 周报 | 每周末自动生成 | 本周交易汇总、周盈亏、新信号统计 |
| 月报 | 每月末自动生成 | 月度绩效、策略贡献、行为分析、因子监控 |
| 自定义 | 用户触发 | 指定时间段的完整绩效分析 |

**可视化要素（至少包含）：**

- 净值曲线（叠加基准）
- 月度收益柱状图
- 盈亏分布直方图
- 持仓占比饼图
- 策略贡献堆叠图

# 第五部分：质量属性

## 15. 非功能性需求

### 15.1 易用性

| 要求 | 说明 |
|------|------|
| 操作步骤 | 核心操作（查看信号、录入交易）≤ 3 步 |
| 术语解释 | 所有专业术语附带悬浮提示或通俗解释 |
| 新手引导 | 首次使用提供向导，帮助完成基础设置 |
| 信号直观性 | L1 用户无需理解指标含义即可根据信号操作 |
| 空状态引导 | 无数据时展示引导文案而非空白页面 |

### 15.2 可扩展性

| 要求 | 说明 |
|------|------|
| 策略插件化 | 新增策略只需实现标准接口，无需修改核心引擎 |
| 策略插件安全 | L3 用户编写的策略插件必须在**受限沙箱环境**中运行（详见下方安全规范） |
| 数据源可替换 | 通过适配器模式接入不同数据服务商（接口契约见附录 D） |
| 多市场预留 | 数据模型中预留市场标识字段，支持未来扩展至 ETF、可转债、港美股 |
| 仓位模型可扩展 | 不同复杂度的仓位模型可并存，用户按层级选择 |

**策略插件沙箱安全规范：**

| 限制项 | 要求 |
|--------|------|
| 数据访问 | 只能通过系统提供的标准数据接口获取数据，禁止直接访问底层存储或内存 |
| 系统调用 | 禁止文件读写（除指定临时目录）、网络请求、进程创建 |
| 执行时限 | 单只标的的策略评分计算 ≤ 100ms；全市场扫描单策略总耗时 ≤ 5 分钟 |
| 内存限制 | 单个策略插件进程内存上限 100MB |
| 隔离方式 | 在受限子进程或容器中执行，崩溃不影响主进程 |
| 审计 | 插件加载、执行、输出均记录审计日志 |

### 15.3 可靠性

| 要求 | 说明 |
|------|------|
| 数据准确性 | 数据入库前必须通过质量校验（见 5.5 节） |
| 信号一致性 | 相同数据 + 相同配置 → 相同信号（幂等性） |
| 数据获取失败 | 自动重试（最多 3 次），失败后告警并暂停当日流水线，不基于不完整数据生成信号 |
| 异常恢复 | 流水线支持从检查点恢复（见下方检查点设计），而非全部重跑 |
| 数据备份 | 用户交易记录和配置定期自动备份 |

**每日流水线检查点设计：**

| 检查点 | 触发时机 | 持久化内容 |
|--------|----------|-----------|
| **CP1：数据就绪** | 数据采集与校验完成 | 当日数据快照版本号、校验结果摘要 |
| **CP2：评分完成** | 全市场评分计算完成 | 全市场综合评分表（含各策略分项得分）、市场状态 |
| **CP3：信号生成完成** | 信号生成与仓位建议计算完成 | 当日信号列表、仓位建议 |

恢复规则：检测到前序检查点已完成时，跳过对应步骤直接从断点继续，避免重复计算。

### 15.4 安全性

| 要求 | 说明 |
|------|------|
| 数据隐私 | 用户交易数据优先存储本地，不上传第三方 |
| 传输加密 | 所有网络传输使用 HTTPS |
| API Key 保护 | 数据源 API Key 加密存储，不明文显示 |
| 操作审计 | 关键操作（交易录入、配置变更）记录审计日志 |

### 15.5 性能

| 要求 | 说明 |
|------|------|
| 日批处理耗时 | 全市场扫描 + 评分 + 信号生成 ≤ 30 分钟 |
| 信号推送延迟 | 信号生成后 ≤ 5 分钟推送到用户 |
| 界面响应 | 页面加载 ≤ 3 秒 |
| 数据存储 | 支持至少 5 年历史数据 + 全部交易记录 |

### 15.6 可观测性

| 要求 | 说明 |
|------|------|
| 运行日志 | 每日批处理全链路日志，包含各环节耗时、处理标的数、信号数 |
| 数据血缘 | 可追溯任一信号的完整计算路径（哪些数据→哪些因子→什么分数→什么权重→最终信号）；**数据血缘是独立的子系统**，需持久化存储每次计算的输入数据版本号、因子快照与信号关联关系，实现成本不可低估。V1.0 范围：Phase 7 实现最小可行版本（信号与当日评分快照绑定）+ Phase 12（V1.0 收尾批次）实现完整因子级溯源（raw_factors / 中性化前后 / 正交化前后 + L1/L2/L3 分层视图）。原推迟 V1.5 计划已于 2026-05-14 升级 V1.0 |
| 异常告警 | 数据缺失、处理异常、流水线超时等情况自动告警 |

### 15.7 技术选型约束条件

本节列出系统对技术实现的隐含约束，供《系统设计文档》技术选型时参考：

| 约束维度 | 要求 |
|----------|------|
| **计算模型** | 全市场评分需支持向量化批量计算（5000+ 标的 × 多因子），技术选型须支持高效数值计算（如基于 NumPy/pandas/Polars 的 Python 生态，或等效方案） |
| **时序数据存储** | 日线行情数据量约为 5000 标的 × 1250 日（5年） = 625 万行；查询模式以"按标的取多日"和"按日期取全市场"为主；选型须支持高效列式访问（如 Parquet 文件 + DuckDB，或列存数据库） |
| **回测与实盘一致性** | 技术架构必须保证回测引擎与实时评分引擎调用同一套计算函数，禁止分别实现 |
| **并行度** | 30 分钟批处理时限要求策略评分支持多标的并行计算；技术选型须支持进程级或线程级并行 |
| **数据血缘存储** | 每日需存储全市场所有标的的因子快照（约 5000 × 因子数 × 每日），需评估存储成本并制定数据保留策略 |

## 附录 C：评分公式汇总

### C.1 策略内评分（以趋势策略为例）

```
trend_score = 0.40 × ma_alignment_score
            + 0.30 × macd_score
            + 0.30 × breakout_score
```

各子因子得分均为横截面 Rank 百分位（0-100）。

### C.2 综合评分

```
composite_score = w_trend × trend_score
               + w_momentum × momentum_score
               + w_mean_rev × mean_reversion_score
               + w_value × value_score
```

其中 `w_*` 取决于当前市场状态（见 7.5 节权重表）。

### C.3 凯利仓位（V1.5）

```
K% = W - (1 - W) / R
实际仓位 = kelly_fraction × K% × 总资产

W = 胜率（历史滚动窗口）
R = 盈亏比（历史滚动窗口）
kelly_fraction = 0.25 ~ 0.50（保守系数）
```
