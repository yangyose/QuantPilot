# QuantPilot（量化领航）系统需求定义

## 1. 项目背景与目标
### 1.1 项目定位
构建一套**中低频、半自动**的股票交易辅助系统，旨在为个人投资者（尤其是缺乏专业金融知识的普通散户）提供专业的交易决策支持，帮助其规范投资行为、提升盈利概率。系统由具备编程背景的开发者发起，未来有望产品化，向更广泛的用户群体推广。

### 1.2 核心价值
- **决策辅助**：基于多策略综合评分，给出清晰的买卖建议和理由。
- **风险管理**：帮助用户科学管理仓位，控制单笔亏损。
- **行为记录**：完整记录交易与决策依据，便于复盘与改进。
- **智能提醒**：关键信号实时推送，不错过交易机会。
- **绩效归因**：自动分析盈亏来源，提升用户认知。

### 1.3 用户画像
- **初期用户（开发者本人）**：具备编程能力，愿意参与系统配置与优化。
- **目标用户（普通散户）**：无专业金融背景，时间有限，希望获得简单明确的交易指导。需要界面友好、操作直观、信号易懂。

## 2. 总体功能模块

### 2.1 账户与资金管理
- **多账户支持**：允许用户创建多个交易账户（如模拟账户、实盘账户），方便独立管理。
- **交易录入**：提供简洁的表单，用户手动录入买卖记录（股票代码、成交价、数量、时间、费用）。系统自动计算持仓、可用资金、盈亏。
- **资产概览**：以图表和数字展示总资产、当日盈亏、持仓市值、可用资金、累计收益率等。
- **资金流水**：记录每一笔出入金、分红、费用，支持筛选和导出。
- **持仓详情**：列表展示持仓股票，包括成本价、现价、盈亏额/率、持仓占比，并可查看个股交易历史。

### 2.2 市场状态识别
- **目的**：动态感知市场整体环境，为策略权重调整提供依据。
- **状态分类**：至少分为三种——**趋势上涨**、**趋势下跌**、**震荡盘整**。可根据需要增加（如强趋势、弱震荡）。
- **识别指标**：基于主要指数（如上证指数、沪深300）的均线排列（如20日与60日均线）、ADX趋势强度、布林带宽度等量化指标，每日计算并输出当前市场状态。
- **可视化**：在首页展示当前市场状态图标及简要说明。

### 2.3 多策略股票筛选与信号合成
#### 2.3.1 内置策略库（可扩展）
系统内置多套经典策略，覆盖不同市场环境。每套策略输出0-100分的得分，并附带判断理由（简要文字）。

- **趋势跟踪策略**：依据均线多头排列、MACD金叉、价格突破近期高点等。理由示例：“股价站上20日均线，MACD金叉，趋势向好。”
- **均值回归策略**：依据RSI超卖、布林带下轨、乖离率等。理由示例：“RSI进入超卖区，股价触及布林下轨，短期反弹概率大。”
- **价值策略**：依据PE/PB历史分位、股息率、盈利增长等（需财务数据）。理由示例：“PE处于历史10%分位，估值偏低，安全边际高。”
- **动量策略**：依据过去3-6个月相对涨幅。理由示例：“近3个月涨幅排名前10%，强势股有望延续。”
- **资金动向策略（可选）**：依据主力资金净流入、北向资金持仓变化、大单资金流向等。理由示例：“近5日主力资金持续净流入，且股价处于相对低位，资金介入迹象明显。”
- **事件驱动策略（可选）**：依据财报超预期、股权激励、回购、重大合同等公告（需舆情数据）。理由示例：“公司发布业绩预增公告，超出市场预期。”

#### 2.3.2 策略权重动态调整
- **权重映射**：预定义在不同市场状态下各策略的权重配置。例如：
    - 趋势上涨：趋势跟踪 50%，动量 20%，价值 10%，均值回归 10%，资金动向 10%。
    - 趋势下跌：价值 40%，均值回归 30%，资金动向 20%，趋势跟踪 5%，动量 5%。
    - 震荡盘整：均值回归 40%，资金动向 30%，价值 20%，趋势跟踪 5%，动量 5%。
- **可配置性**：允许用户（或管理员）自定义权重映射，以适应个人偏好。

#### 2.3.3 综合评分与股票池
- **全市场扫描**：每日收盘后，对全部A股（或自选股池）运行所有策略，得到各策略得分。
- **综合评分**：按当前市场状态下的权重，加权计算每只股票的最终综合评分（0-100分）。
- **目标股票池**：按综合评分排序，选出前N只作为“关注池”（N默认20，可配置）。同时展示每只股票在各策略下的得分和理由。
- **黑白名单**：支持用户手动添加自选股、排除某些股票（如ST股、黑名单）。

### 2.4 买卖信号生成
- **买入信号**：当关注池中的某只股票综合评分超过**买入阈值**（默认80分，可配置），且当前无持仓（或符合加仓规则）时，触发买入信号。信号包含：
    - 股票名称代码、当前价格。
    - 综合评分及主要贡献策略。
    - 策略理由摘要（如“趋势跟踪得分95：均线多头排列；资金动向得分85：主力连续净流入”）。
    - 建议买入价区间（如现价±1%）。
    - 参考止损位（基于策略或固定比例，如-8%）。
    - 参考仓位（基于固定比例仓位模型，如“建议买入金额：总资产的10%”）。
- **卖出信号**：当持仓股票的综合评分低于**卖出阈值**（默认40分，可配置），或特定策略发出止损信号（如趋势破位），触发卖出信号。信号包含：
    - 股票名称代码、当前价格。
    - 综合评分及下降原因。
    - 主要卖出理由（如“趋势跟踪得分降至20：股价跌破60日均线；资金动向得分降至30：主力开始流出”）。
    - 建议卖出价区间。
- **信号更新**：每日收盘后重新计算评分，如有新信号产生，立即更新界面并触发提醒。

### 2.5 仓位管理建议
- **核心原则**：为普通散户提供简洁、科学的仓位建议。
- **固定比例法**：默认每笔交易占用总资金的固定比例（如10%）。若账户总资产10万元，则单只股票建议买入金额为1万元。用户可自定义该比例。
- **加仓规则**：初始仅建议一笔买入。若后续策略再次发出强烈买入信号（如综合评分仍高于买入阈值且持仓盈利），可提示加仓，但总持仓不超过总资金的20%。
- **止损提示**：根据策略逻辑或固定比例（如-8%）计算止损价，并在信号中明确提示。例如：“建议止损价：XX元（较买入价下跌8%）”。

### 2.6 信号提醒与交互
- **提醒方式**：
    - **界面内通知**：系统首页显著位置展示最新信号列表。
    - **微信推送**：仅当产生买入或卖出信号时，通过微信发送提醒。消息内容简洁明了，包含股票名称、代码、信号类型、简要理由、建议操作。
- **交互要求**：用户点击信号可跳转至详情页，查看完整的策略评分、历史信号、走势图（可选）。
- **交易录入便捷性**：在信号详情页提供“一键录入”入口，快速填写交易记录（股票代码、价格自动填充，用户只需确认数量和时间）。

### 2.7 绩效归因与复盘
- **自动报告**：每周、每月自动生成绩效报告。
- **归因维度**：
    - **收益来源**：总收益、收益率、最大回撤、夏普比率。
    - **选股贡献**：按信号买入的股票整体表现与大盘对比。
    - **策略贡献**：如果用户跟随某策略的信号进行交易，该策略的累计收益、胜率、盈亏比（通过信号回溯）。
    - **交易行为分析**：平均持仓天数、交易频率、盈亏分布。
- **可视化**：使用图表展示净值曲线、月度收益柱状图、盈亏分布饼图等，便于理解。

### 2.8 用户设置与配置
- **基础设置**：账户初始资金、交易费用率、买入/卖出阈值、固定仓位比例等。
- **策略配置**：启用/禁用具体策略，调整各策略参数（如均线周期、RSI周期），自定义市场状态权重映射。
- **提醒设置**：是否开启微信提醒，设置提醒时间段（如仅交易时段）。
- **数据源设置**：选择数据源（需支持后续扩展），输入API密钥等。
- **黑白名单管理**：添加自选股、排除股。

### 2.9 数据需求
- **基础行情**：A股日线数据（开高低收、成交量、复权因子），历史数据需覆盖至少3年。
- **财务数据**：季度/年度PE、PB、ROE、净利润增长率、股息率等（用于价值策略）。
- **指数数据**：主要指数（上证、深证、沪深300、创业板指）日线数据。
- **资金流向数据（可选）**：如主力资金净流入、北向资金持仓、大单资金比例等，需对接相应数据服务商。
- **舆情/公告（可选）**：如业绩预告、重大合同、增减持、股权激励等，可通过第三方API接入。

## 3. 非功能性需求

### 3.1 易用性
- **界面简洁**：符合普通用户习惯，避免专业术语过多，或用通俗解释。
- **操作流畅**：交易录入步骤不超过3步。
- **引导式体验**：首次使用提供向导，帮助用户完成基础设置。

### 3.2 可扩展性
- **策略插件化**：策略模块设计为可插拔，便于日后增加新策略。
- **数据源适配**：支持切换不同的数据服务商，降低对单一数据源的依赖。
- **多市场支持**：架构上预留港股、美股等扩展可能。

### 3.3 可靠性
- **数据准确性**：必须确保基础数据正确，避免因数据错误导致错误信号。
- **信号一致性**：相同数据下信号应稳定可重现。
- **错误处理**：数据获取失败时应有重试机制，并提示用户。

### 3.4 安全性
- **数据隐私**：用户交易数据存储在本地或用户可控的云端，不泄露。
- **账户安全**：若涉及云服务，需加密传输与存储。

## 4. 未来可扩展方向
- **模拟交易**：自动根据信号模拟执行，对比用户实盘操作。
- **更多交易品种**：ETF、可转债、港股通。
- **社交功能**：允许用户分享策略或信号（需谨慎）。
- **移动端APP**：在网页版基础上开发原生App或小程序。
